# Flow Board 任务管理系统

一个基于 Vue 3 + Vite + TypeScript 实现的任务管理系统，面向课程作业场景设计，支持任务新增、编辑、删除、状态切换、搜索筛选、排序、统计分析和本地持久化。界面采用多卡片仪表盘风格，支持响应式布局与主题切换。

## 项目简介

本项目用于解决学习、工作和生活任务碎片化、难追踪的问题。系统围绕“记录任务、规划时间、跟踪进度、查看完成情况”四个核心目标展开，提供以下能力：

- 用统一的数据结构管理任务标题、描述、状态、优先级、截止日期、分类和标签
- 通过任务列表页完成实时搜索、筛选、排序和状态切换
- 通过统计页查看总任务数、完成率、逾期任务和近 7 天完成趋势
- 使用 localStorage 做本地持久化，刷新后数据仍然保留
- 支持导入与导出 JSON 备份，便于演示和迁移数据

## 页面说明

- 仪表盘：展示完成率、任务统计、近期更新和即将到期任务
- 任务列表页：展示全部任务，支持搜索、筛选、排序、详情查看和状态切换
- 看板视图页：通过拖拽方式在待办、进行中、已完成之间移动任务
- 日历视图页：通过月历查看每天的到期任务，并聚焦选中日期任务清单
- 新增 / 编辑任务页：用于创建或修改任务信息
- 统计分析页：使用 ECharts 展示完成趋势、分类占比和优先级分布
- 设置页：提供主题切换、导入导出、清空数据等功能

## 项目运行步骤

### 1. 安装依赖

如果你的终端已经配置好 Node.js 环境变量，可以直接执行：

```bash
npm install
```

如果 VS Code 终端提示找不到 npm，可使用本机绝对路径执行：

```powershell
C:\Program Files\nodejs\npm.cmd install
```

### 2. 启动开发环境

```bash
npm run dev
```

如果终端仍然识别不到 npm，可执行：

```powershell
C:\Program Files\nodejs\npm.cmd run dev
```

### 3. 生产构建

```bash
npm run build
```

## 数据持久化方案说明

本项目采用 localStorage 本地持久化方案。

- 任务数据保存在浏览器本地 storage 中
- 新增、编辑、删除、状态变更后立即自动保存
- 刷新页面后数据不会丢失
- 支持导出 JSON 备份文件
- 支持重新导入 JSON 恢复任务数据
- 设置页支持清理已完成任务和清空全部数据
